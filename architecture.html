interfaces/

Custom msg / srv / action definitions
  • msg/ToolStatus.msg — tool_id, slot, status, ts

  • msg/PLCState.msg — led_color, led_mode
  • srv/CheckToolFeasibility.srv
  • srv/UpdateToolStatus.srv
  • action/UnitAction.action

voice/

STT + Gemma 4 intent node
  • whisper_node.py → /voice/raw_text
  • gemma_intent_node.py
  •   input: raw_text + DB snapshot
  •   output: /voice/intent
  •     {action, tool_id, feasible, reason}
  • wake_word_detector.py (optional)

vision/

Perception pipeline (shared logic)
  • yolo_node.py → /vision/detections
  • pose_node.py → /vision/tool_poses
  • tracker_node.py → /vision/tracked
  • context_builder.py → /vision/scene_json
  •   (feeds Track C over topic)

orchestrator/

BT + unit_action_server
  • behavior_manager.py (BT tick loop)
  • bt_nodes/fetch_tool.py
  • bt_nodes/return_tool.py
  • bt_nodes/handover.py
  • bt_nodes/recovery.py
  • unit_action_server.py

db/

Database interface + FOD management
  • db_node.py (ROS2 service server)
  • tool_inventory.py (CRUD)
  • fod_monitor.py
  •   out > timeout → 'missing' alert
  •   missing → PLC yellow flash
  • event_logger.py (audit trail)

motion/

Motion control + handover detection
  • dsr_controller.py (Track A)
  • rl_policy_node.py (Track B)
  • grasp_planner.py
  • handover_detector.py
  •   F/T + vision → HandoverEvent
  •   → DB log + PLC update

plc/

PLC communication + LED state
  • plc_node.py (Modbus/EtherNet IP)
  • led_state_mapper.py
  •   system_state → LED command
  • plc_watchdog.py
  •   PLC silent > 2s → warning log

  • gripper_actions.py → grasp, release
  • handover_actions.py → handover_wait

hal/ — no ROS2

  • arm_driver.py (DSR / RL backend)
  • gripper_driver.py
  • ft_sensor.py
  • db_client.py (direct SQL)

  • imports: unit_actions, hal, perception
  • imports: db_client, plc_client
